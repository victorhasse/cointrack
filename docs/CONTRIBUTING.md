# Contributing to Cointrack

First of all, thank you for your interest in contributing! Open-source projects thrive thanks to people like you.

To maintain code quality and project organization, please follow these guidelines:

## How can I contribute?

1.  **Reporting Bugs:** Open an Issue detailing the error, the expected behavior, and the steps to reproduce it.
2.  **Suggesting Features:** We love new ideas! Describe how the feature would benefit users.
3.  **Submitting Pull Requests (PRs):**
    *   Fork the project.
    *   Create a branch for your modification (`git checkout -b feature/my-improvement`).
    *   Keep the code clean and follow the project's standards.
    *   Submit the PR with a clear description of what was changed.

## Environment Setup

1. Clone your fork: `git clone https://github.com/victorhasse/cointrack.git`
2. Install the dependencies (ex: `npm install` ou `pip install -r requirements.txt`).
3. Ensure that all tests are passing before submitting.

## Code of Conduct

Be respectful and collaborative. Constructive discussions are always welcome; toxicity is not.
