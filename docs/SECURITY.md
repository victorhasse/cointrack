# Security Policy

Data and transaction security at **Cointrack** is our top priority. If you find a vulnerability, we appreciate you letting us know in a responsible manner.

## Supported Versions

Currently, we provide security updates for the following versions:

| Version | Supported |
| :--- | :--- |
| v1.x | ✅ Yes |
| < v1.0 | ❌ No |

## Reporting a Vulnerability

**Please do not open a public Issue to report a security flaw.**

If you have discovered a vulnerability, please follow these steps:
1. Send an email to victor@victorhasse.com (or use GitHub's "Report a vulnerability" system, if available).
2. Include technical details, a proof of concept (PoC), and the potential impact.
3. Give us a reasonable amount of time to respond and fix the flaw before any public disclosure.

## Our Promise

*   We will respond to your report within 48 business hours.
*   We will keep you informed about the progress of the fix.
*   We will give proper credit if the vulnerability is confirmed and resolved.

**Note:** Since this is a crypto-focused project, never share your private keys or API secrets in error logs or support communications.
